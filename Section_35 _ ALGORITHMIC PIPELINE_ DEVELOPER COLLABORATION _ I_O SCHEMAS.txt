### 35.1 Executive Summary of Developer Collaboration
This system represents a comprehensive adaptive feedback engine for Algerian university EFL students. The system covers three integrated dimensions: Writing (NLP), Behavior (Temporal analysis across 8 assignments), and Activity (Moodle logs). 
Work is divided into 25 algorithms, distributed among three developers (7 algorithms each) plus an integration phase (4 algorithms). Each developer operates independently on their specific dimension, producing a unified JSON output schema.

*Note: The KMeans algorithm was excluded and replaced with Agglomerative Clustering, as it is non-parametric and more appropriate for a small sample size (N=50). Additional ML techniques integrated based on Youcef's suggestions include: TF-IDF vectorization, Cosine similarity, PCA, and Cohen's d effect size.*

| Developer | Dimension | Algorithms | Scope of Work | Primary Output |
|---|---|---|---|---|
| **Adnan** | Writing (NLP) | 7 Algorithms | Linguistic and argumentative analysis of student texts. | Writing Score (0-20) + L1/NLP markers. |
| **Youcef** | Behavior & Temporal | 7 Algorithms | Analyzing temporal trajectories across 8 assignments and diagnosis. | Student Classification + Triggered Rules + Temporal Trajectory. |
| **Abdelkader** | Moodle Activity | 7 Algorithms | Parsing Moodle log records. | Behavioral Flags + Sessions + Activity Patterns. |
| **Integration** | System Output | 4 Algorithms | Merging all outputs into a final decision. | Final Grade + Risk + Diagnosis + Feedback. |

### 35.3 Module 2: NLP Feature Extraction & Writing Analysis (Developer: Adnan)
- **Scope of Work:** Responsible for all NLP and argumentative analysis of student texts. This includes linguistic tokenization, extracting 6 deterministic features, rubric evaluation, L1 transfer error detection (Arabic/French), TF-IDF vectorization, Cosine Similarity (plagiarism/similarity), and Semantic embeddings (Sentence-BERT).

#### Adnan's 7 NLP Algorithms
| # | Algorithm | Function | Precise Output |
|---|---|---|---|
| **1** | spaCy (`en_core_web_sm`) | POS + dependency parsing, tokenization. | `tokens[]`, `sentences[]`, `POS_tags[]`, `dep_parse` |
| **2** | 6 Deterministic Features | Calculate TTR, MTLD, Cohesion, AVD, Error Density, Arg Markers. | Numeric values + interpretation (weak/moderate/strong). |
| **3** | Regex + spaCy rubric | Evaluate 22 criteria across 4 sections (intro/body/counter/conclusion). | `rubric_scores{criterion: 1-5}`, `section_grade (0-5)`, `total_grade (0-20)`. |
| **4** | L1 Pattern Matching | Detect Arabic/French transfer patterns. | `l1_errors[]{type, text, position}`, `l1_error_count`, `dominant_l1_source`. |
| **5** | TF-IDF Vectorization | Represent text as a sparse vector (`sklearn TfidfVectorizer`). | `tfidf_vector (sparse)`, `feature_names[]` (Saved as `.npz`). |
| **6** | Cosine Similarity | Calculate similarity between every essay and all others. | `similarity_matrix (NxN)`, `top_3_similar_essays[]`. |
| **7** | Sentence-BERT (frozen) | Semantic embedding without fine-tuning (768-dim). | `embedding_vector[768]`, `semantic_search_index` (Saved as `.npy`). |

### 35.4 Module 3: Temporal Trajectory & Behavior Inference (Developer: Youcef)
- **Scope of Work:** Responsible for analyzing the student's evolution across the 8 assignments and diagnosing macro-behavioral patterns. This involves calculating trajectories, normalizing metrics to percentiles, conducting non-parametric statistical tests, calculating effect sizes, clustering students with Agglomerative Clustering, and firing the 18 pedagogical rules.

*Note: Youcef relies directly on the outputs of Adnan (NLP linguistic features per essay) to compute the temporal trajectories.*

#### Youcef's 7 Temporal & Behavioral Algorithms
| # | Algorithm | Function | Precise Output |
|---|---|---|---|
| **1** | Descriptive OLS Regression | Calculate the slope of evolution per student across 8 tasks. | `trajectory_slopes{word_count, error_density, cohesion, grade}`, `trend_direction`. |
| **2** | Percentile Normalization | Convert features to percentile ranks (0-100) across the cohort. | `percentiles{feature: 0-100}` for every student and task. |
| **3** | Mann-Whitney U + Kruskal-Wallis | Non-parametric tests to compare student risk groups. | `group_comparisons[]{features, U_stat, p_value, effect_size_r}` (Global). |
| **4** | Spearman Correlation | Rank correlation between behavior and writing features. | `correlation_matrix (NxN)`, `significant_pairs[]` (Global). |
| **5** | Cohen's d Effect Size | Measure magnitude of difference between groups (e.g. high vs low risk). | `effect_sizes[]{group1, group2, feature, d_value, interpretation}` (Global). |
| **6** | Agglomerative Clustering (k=2-4) | Descriptive grouping of students. (Replaced KMeans). | `cluster_id (0-3)`, `cluster_label`, `dendrogram_path` (Saved as PNG). |
| **7** | 18 Diagnostic Rules | If/Then pedagogical rules covering 4 categories (process, product, revision, engagement). | `triggered_rules[]{rule_id, name, interpretation, confidence, priority}`. |

#### Technical & Acceptance Criteria
- **Tech Stack:** Python 3.11+, `scipy.stats` (Mann-Whitney, Kruskal-Wallis, Spearman), `scikit-learn` (Agglomerative Clustering), `matplotlib` (Dendrograms), `numpy`.
- **Why Agglomerative over KMeans?** (1) Fully non-parametric, estimating zero parameters from the data. (2) Does not require pre-defining `k`; the dendrogram can be cut at any level. (3) The dendrogram visually helps understand hierarchical cluster structures, whereas KMeans assumes spherical clusters.
- **Acceptance:** 
  - Complete JSON generated for all 50 logged students containing their 4 temporal trajectories.
  - Every student mapped to a cluster label (0-3).
  - List of triggered diagnostic rules accurately generated per essay.
  - 4 competence scores (0-100) calculated per student.
  - A visual PNG Dendrogram representing the cluster structure is saved.
  - Full statistical hypothesis testing matrix (Mann-Whitney + Spearman) completed.

### 35.5 Module 4: Final Integration (All Developers)
- **Scope of Work:** The integration is a joint effort between Adnan, Youcef, and Abdelkader. After each developer completes their respective 7 algorithms, they submit their outputs in a unified JSON format to the Integration Module. This module applies 4 collaborative algorithms to fuse the data into the final system output.

#### The 4 Collaborative Integration Algorithms
| # | Algorithm | Inputs | Precise Output |
|---|---|---|---|
| **1** | Fixed-Weight Fusion (70/30 Baseline) | Rubric Grade (from Adnan) + NLP Composite (from Adnan). Note: 70/30 weights are initial heuristic baselines subject to empirical calibration via Gold Standard Corpus. | `final_percentile = 0.70*rubric + 0.30*nlp_composite` → `final_grade` (0-20). |
| **2** | Weighted Evidence Accumulation | Triggered Rules (Youcef) + Behavioral Flags (Abdelkader). | `alert_score` (Sum of priority*confidence) + `alert_level` (low/moderate/high). |
| **3** | Bayesian Inference (Literature Priors) | Multiple metrics across dimensions + priors from SLA literature. | Posterior probabilities like `P(argument_weak \| evidence)`, `P(cohesion_weak \| evidence)`. |
| **4** | Decision Tree (Fixed Structure) | `risk_band` + `rules`. Fixed structure for feedback generation. | `feedback_type`, `feedback_template_id`, `feedback_text`. |

#### Integration Workflow & Directory Structure
The operational sequence follows this strict pipeline:
1. **Adnan:** Extracts linguistic and argumentative features for each text.
2. **Abdelkader:** Extracts Moodle activity and behavioral metrics (runs in parallel with Adnan).
3. **Youcef:** Uses Adnan's outputs to calculate temporal trajectories and behavior clusters.
4. **Integration (All Three):** Merges all outputs via the 4 integration algorithms to compute `FINAL_GRADE`, `RISK`, and `FEEDBACK`.
