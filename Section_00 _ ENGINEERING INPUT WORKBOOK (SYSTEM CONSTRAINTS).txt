Based on the explicit "Engineering Input Workbook", the fundamental guardrails for programming the system are as follows:

- Purpose: Specifies the minimum indicator dataset needed to program an AI-supported adaptive feedback engine for 60 EFL academic writing students.
- Unit of Analysis: One row in the database = (One Student) × (One Feedback Loop) × (One Draft Pair).
- Core Pipeline: Raw data → Calculated indicators → Threshold bands → Learner state → Rule ID → Feedback template → Teacher validation.
- Primary Data Sources: Analytic rubric scores, NLP textual features, Moodle engagement traces, revision behaviour, feedback uptake, help-seeking categories, and teacher validation.
- Engineer Note: The system mathematically computes all intermediate grey-logic formulas. The only human/external inputs are the raw text and Moodle trace logs.
- Privacy Guardrail: Use anonymous IDs only. Never ingest names, photos, IP addresses, or raw private messages into the adaptive decision engine.
- Implementation Mandate: Store each feedback loop as a "Transaction Record". Do not overwrite old rows. The longitudinal Learner Model must dynamically update after each new draft pair.
